# Winnsen Product Canvas

面向 Winnsen 自助终端、智能柜和售卖机业务的无限画布原型。

## 当前版本

- 产品画布：把 Parcel Locker、Laundry Locker、Flower Vending、Charging Locker 等产品卡放在同一个空间里。
- 客户需求：记录客户项目、区域市场、数量、目标价、交期、定制需求和版本差异。
- 版本差异：把 V0 需求版、V1 方案版等差异项沉淀到需求卡，便于报价和评审。
- 客户项目库：内置典型报价前项目模板，覆盖客户、区域、产品线、数量、目标价、交期、定制项、风险和所需资料。
- 报价前评审：可从客户项目一键生成需求卡或完整项目包，自动包含产品卡、需求卡、风险评审、界面流程、素材缺口和视频计划。
- 项目地图导出：可把客户项目库生成画布地图，或导出 Markdown 进入报价前会议和产品评审。
- 报价决策门：按客户场景、产品适配、配置差异、软件/支付、结构/安装、合规、素材和交期给项目打分，判断可报价、补资料后报价、先技术评审或暂缓报价。
- Gate 导出：报价决策门可生成画布品审节点，并进入评审包与 Markdown 导出。
- 品审节点：记录结构、门锁、屏幕、线束、防水、UI 流程、客户定制等评审点。
- 界面流程：按产品线生成取件、洗衣投递/取回、售卖购买、充电租借等屏幕流程，包含入口方式、关键步骤、异常状态和界面检查点。
- 品审模板库：内置柜体钣金、门板/铰链/锁具、屏幕操作区、电控线束、装配售后、使用环境与认证等模板。
- 整套品审清单：可从当前产品卡一键生成完整结构/产品品审节点，并自动连线到画布。
- 知识沉淀：把评审结论转成可复用经验卡。
- 知识库沉淀：可把当前品审或全部品审批量转成结构学习卡，并导出知识库 Markdown。
- 图库 / 模型库：把产品图、安装图、渲染图、3D 模型、UI 截图作为项目素材管理。
- 素材入库：记录素材负责人、版本、存储路径、来源、审批状态、必备素材和缺口清单。
- 素材缺口：可从当前产品或全部产品一键生成素材需求包，覆盖产品图、结构图、UI 截图、3D 模型、客户场景和视频镜头。
- 文件入库台账：把素材包拆成文件级清单，记录文件名、类型、格式、负责人、路径、状态、用途和公开等级。
- 项目文件包：可把客户项目库中的所需资料汇总成文件总台账，用于报价前资料准备、素材审核和视频生成。
- 开源参考图谱：把 tldraw、Excalidraw、AFFiNE、AppFlowy 的可借鉴能力映射到 Winnsen 产品工作流。
- GitHub 参考证据台：把 tldraw、Excalidraw、AFFiNE、AppFlowy、draw.io 的星标、license、活跃信号、成熟能力和 Winnsen 采纳决策变成可评审证据卡。
- 产品路线图：按 P0-P4 规划从静态原型、真实资料、画布引擎、AI 工作流到团队产品 OS 的落地阶段。
- 团队角色矩阵：定义产品设计、界面设计、产品品审、图库模型、视频生成、销售报价、软件集成和管理评审的权限、输入、输出和交接对象。
- 团队交接流程：把客户机会、报价 Gate、产品品审、UI/软件、素材文件、视频方案和知识库复盘串成报价前协作闭环。
- 数据模型总览：把工作区、画布节点、产品资料、客户项目、报价 Gate、品审、UI 流程、素材文件、知识卡、视频计划和权限转成可开发 schema。
- MVP 开发规格：按 P0-P3 定义无限画布核心、产品资料库、报价 Gate、知识库、素材台账、AI 视频、团队权限和外部集成的范围与验收标准。
- 工作流启动器：按真实任务一键启动产品设计冲刺、报价前评审、UI 交付、素材视频准备、知识复盘和开发规划。
- 今夜执行目标：仿照 tldraw、Excalidraw、AFFiNE、AppFlowy、draw.io 的成熟产品骨架，再叠加 Winnsen 产品设计、界面设计、品审、报价 Gate、图库模型库、视频生成和知识库特色。
- 苹果界面风格：采用系统字体、浅灰背景、半透明玻璃面板、细边框、柔和阴影、蓝色主操作和 macOS/iPadOS 式浮层质感。
- 命令中心：用 Apple 风格浮层集中常用动作，可搜索并直接生成资料包、补齐台、试用包、会话记录、反馈迭代和评审报告。
- 试用启动台：用 Apple 风格进度条、下一步卡片和一键 recipe 生成内部试用工作区，自动铺开 GitHub 参考证据、仿照矩阵、官网资料种子、资料补齐、试用包、会议闭环和反馈迭代。
- 角色试用路线：按产品设计、界面设计、产品品审、视频生成、知识库管理、图库模型库 6 类内部角色生成专属试用路径、产出和验收标准。
- 试用验收中心：用 Gate 方式检查成熟产品骨架、Apple 风格、试用启动、角色路线、真实资料、会议反馈和报告导出是否具备内部试用条件。
- 现场反馈：内部同事试用时可按角色、场景、优先级、评分和观察记录真实反馈，并进入画布、评审包和 Markdown 导出。
- 明早晨会包：把过夜迭代后的 Gate、现场反馈、资料缺口、风险雷达、发言顺序和负责人跟进自动汇总成可导出的晨会材料。
- 执行派单：把晨会风险、现场反馈、Gate 评分、资料缺口和 backlog 自动转成 P0/P1 任务节点、负责人、截止时间和验收输出。
- 内部试用包：定义新品设计冲刺、报价前评审、UI 交付、素材入库和知识复盘的试用场景、60 分钟会议流程与验收 Gate。
- 试用起步导航：按 6 步引导内部同事选择路线、选择对象、启动工作流、检查缺口、导出评审包并记录反馈。
- 试用会话记录：把内部试用会议的范围、决策、证据、行动项、负责人和验收标准沉淀回画布，并导出会后纪要。
- 试用反馈 / 迭代：记录内部试用观察、Gate 评分、影响、响应动作和下一轮开发优先级。
- 产品资料库：内置 Winnsen 业务相关产品资料字段，包括型号、产品线、应用场景、门数/结构、屏幕、支付、软件集成、材料和认证。
- 产品线筛选：可按 Parcel、Laundry、Flower、Refrigerator、Mini Mart、Luggage、Tool/Safety 等产品线筛选资料库。
- 产品组合画布：可把当前筛选的产品线一键生成产品组合地图，用于产品规划、报价前方案选择和任务拆分。
- 产品库导出：可把当前筛选的产品资料导出为 Markdown 表格，进入会议纪要、知识库或报价前清单。
- 资料包生成：可从产品资料库一键生成产品卡、品审点、素材需求和视频脚本节点。
- 客户 brief 生成：可从当前产品或全部产品一键生成客户需求卡，并导出客户需求 Markdown。
- CSV 批量导入：支持把产品资料表导入为画布产品卡，可识别型号、产品线、场景、门数、材料、屏幕、支付、软件、认证、模块和任务。
- CSV 项目启动：导入时可自动生成整套结构品审清单、界面流程和素材需求节点，把产品表直接转成项目画布。
- 官网资料种子包：基于 Winnsen 官网公开产品标题生成 10 个产品资料种子，可一键生成产品卡、品审、UI 流程和素材需求，后续再由内部团队补齐真实型号、图片、结构图和认证资料。
- 内部资料补齐台：把官网种子产品拆成产品资料、结构/BOM、UI/软件、图库模型、视频素材、报价认证 6 类内部补齐任务，并生成责任矩阵和 Markdown 补齐表。
- 素材 / 模型库：内置图片、PDF、3D、视频素材包入口，可一键添加素材卡。
- 视频生成：从产品资料、界面流程和评审结论生成产品介绍视频脚本、镜头、旁白和 prompt。
- 本地保存：节点内容、位置、缩放视图会保存到浏览器 localStorage。
- 节点编辑：右侧面板可修改标题、说明、状态、参数和备注。
- 关联生成：可从当前节点直接生成关联品审点或视频分镜卡。
- 导出：可导出当前画布 JSON，后续可接入数据库或 AI 工作流。
- 评审包导出：评审包支持 JSON 和 Markdown，便于进入知识库、会议纪要或团队评审流程。
- 评审包客户维度：评审包会汇总客户需求、版本差异、界面流程、品审、素材和视频计划。
- UI 清单导出：界面流程可单独导出 Markdown，用于交给 UI 设计、软件、结构和销售团队对齐。
- 素材库导出：图库/模型库可单独导出 Markdown，用于整理素材目录、3D 模型、视频镜头和待补缺口。
- 导入：支持导入已导出的 JSON 工作区。
- 任务看板：从产品卡和品审点自动汇总 P1/P2/P3 待办。
- AI 工作台：可根据当前产品及关联节点生成产品设计、界面设计、结构品审、素材整理、知识沉淀和视频脚本提示词。
- AI Prompt 包：可把整套提示词生成画布知识卡，或导出 Markdown 给 ChatGPT、视频生成工具、设计工具和知识库使用。
- Winnsen 品审维度：内置柜体、门/铰链/锁、屏幕操作区、线束、钣金、装配、维护等检查点。
- 方案模板：一键生成新品结构品审、客户定制方案、视频素材规划三类画布群组。
- 结构学习卡：评审包内自动生成产品结构学习卡，便于沉淀经验。
- Apple 风格界面：采用浅色磨砂面板、系统蓝主色、细边框、柔和阴影、圆角卡片和 macOS/iPadOS 风格工作台布局。

## 打开方式

直接打开 `index.html` 即可运行原型。

CSV 样例在 `samples/winnsen-product-catalog-sample.csv`，页面右侧“产品资料库”里也可以直接下载样例。

## 下一阶段

1. 接入真实产品资料库：产品型号、门数、材质、屏幕尺寸、支付方式、软件集成、证书。
2. 接入本地或云端素材存储：图片、视频、PDF、3D 模型、UI 截图。
3. 评估画布引擎：参考 tldraw 的自定义节点、协作和运行时 API，同时保留 Excalidraw 的轻量导出体验。
4. 把数据模型转成正式数据库和接口：Workspace、CanvasNode、ProductProfile、CustomerProject、QuoteGate、AssetFile、KnowledgeCard、RolePermission。
5. 接入 AI：产品品审、竞品分析、视频脚本、销售文案、知识库问答，并把 AI Prompt 包升级为可直接调用模型的工作流。
6. 接入 3D 预览：用 Three.js 浏览柜体模型和结构拆件。
7. 接入团队协作：评论、任务、版本、权限、评审报告导出，并把团队角色矩阵转成真实账号权限。
8. 增加导入 JSON、批量产品数据表和供应商/客户版本记录。
